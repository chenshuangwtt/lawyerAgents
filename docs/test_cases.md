# 多领域法律咨询测试用例

用于验证 LangGraph 多域并行检索系统的测试问题集。
按涉及领域数量分类，每个问题标注了预期命中的领域和对应的法律依据。

---

## 配置

### 多域协作

系统通过 `MULTI_DOMAIN_MAX_DOMAINS` 控制多域并行检索的最大领域数。

| 位置 | 说明 |
|------|------|
| `.env` | 设置 `MULTI_DOMAIN_MAX_DOMAINS=N` 覆盖默认值 |
| `app/config.py` | 默认值 `3` |
| `app/classifier.py` | LLM 输出的领域超过上限时，按相关度排序取前 N 个 |
| `app/law_registry.py` | 提示词限制 LLM 最多输出 3 个领域 |

**生效方式**：修改 `.env` 后重启后端服务即可，无需改代码。

**成本考量**：每个领域并行执行一次完整检索（BM25 + 向量 + Rerank），领域越多 token 消耗越高，响应延迟取决于最慢的分支。建议取值 2\~4。

### 三项优化（默认关闭）

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| `ENABLE_WEIGHTED_MERGE` | `false` | 多域加权合并：按领域优先级排序结果（如刑事优先于行政） |
| `DOMAIN_PRIORITY_ORDER` | `刑事,行政,治安,监察` | 领域优先级顺序（逗号分隔） |
| `ENABLE_INTELLIGENT_EXPANSION` | `false` | 上下文智能拓展：LLM Sub Agent 判断候选条文相关性 |
| `EXPANSION_DEPTH` | `1` | 拓展深度：0=关闭 / 1=标准 / 2=深度 |
| `ENABLE_SEMANTIC_VERIFICATION` | `true` | 引用语义溯源：标注置信度 + 检测遗漏引用 |
| `ENABLE_CASE_RETRIEVAL` | `false` | 案例检索默认隐藏；开启后仅展示高相关官方案例 |
| `CASE_USE_SEMANTIC` | `true` | 案例语义检索（LanceDB，首次启动自动构建向量库） |
| `CASE_TOP_K` | `3` | 案例检索返回条数 |

案例检索会按领域和核心事实关键词做强相关过滤。当前官方精选案例库规模较小，默认不在主流程展示案例；需要测试案例检索时，手动开启 `ENABLE_CASE_RETRIEVAL=true`，且只有高相关案例才会返回。

这些优化开关独立，可单独开启测试。关闭时行为与原版主流程保持一致。语义溯源默认开启，案例检索默认隐藏。

## 单领域

### 1. 劳动
**问题：** 试用期被无故辞退，公司需要赔偿吗？能赔多少？
**预期领域：** 劳动
**相关法律：** 劳动合同法

### 2. 婚姻
**问题：** 离婚后孩子抚养权怎么判？法院主要考虑哪些因素？
**预期领域：** 婚姻
**相关法律：** 民法典

### 3. 刑事
**问题：** 盗窃罪的立案标准是多少？偷多少钱才构成犯罪？
**预期领域：** 刑事
**相关法律：** 刑法

### 4. 民事诉讼
**问题：** 去法院起诉需要准备哪些材料？诉讼费怎么算？
**预期领域：** 民事诉讼
**相关法律：** 民事诉讼法

### 5. 网络与数据
**问题：** APP 要求读取通讯录才给用，这合法吗？能拒绝吗？
**预期领域：** 网络与数据
**相关法律：** 个人信息保护法

### 6. 税务
**问题：** 个人所得税的专项附加扣除有哪些？怎么申报？
**预期领域：** 税务
**相关法律：** 增值税法、个人所得税法

### 7. 治安
**问题：** 在小区聚众赌博被抓获，会被拘留多少天？罚款多少？
**预期领域：** 治安
**相关法律：** 治安管理处罚法

### 8. 商事
**问题：** 公司商业秘密被竞争对手窃取，能要求多少赔偿？
**预期领域：** 商事
**相关法律：** 反不正当竞争法

## 双领域

### 1. 劳动 + 刑事
**问题：** 公司高管挪用了员工的工资款，员工该怎么维权？能追究刑事责任吗？
**预期领域：** 劳动、刑事
**相关法律：** 劳动合同法、刑法
**说明：** 主领域应为劳动，核心诉求是追讨工资/劳动报酬；刑事作为副领域，结合事实判断可能涉及挪用资金、职务侵占或拒不支付劳动报酬等线索。不得误判为合同。

### 2. 婚姻 + 网络与数据
**问题：** 离婚诉讼中，配偶偷拍的聊天记录和开房记录能作为出轨证据吗？
**预期领域：** 婚姻、网络与数据
**相关法律：** 民法典、个人信息保护法
**说明：** 证据的合法性判断涉及隐私权

### 3. 治安 + 刑事
**问题：** 被人殴打成轻微伤，对方会被治安拘留还是判刑？
**预期领域：** 治安、刑事
**相关法律：** 治安管理处罚法、刑法
**说明：** 轻微伤走治安处罚，轻伤以上走刑事追诉

### 4. 未成年人 + 刑事
**问题：** 15 岁少年在校打架把同学打成重伤，要负刑事责任吗？家长要赔偿吗？
**预期领域：** 未成年人、刑事
**相关法律：** 未成年人保护法、刑法
**说明：** 刑事责任年龄 + 监护人赔偿责任

### 5. 食药安全 + 民事诉讼
**问题：** 在餐厅就餐后食物中毒住院，该怎么起诉索赔？需要哪些证据？
**预期领域：** 食药安全、民事诉讼
**相关法律：** 食品安全法、民事诉讼法
**说明：** 食品安全责任 + 举证与诉讼流程

### 6. 商事 + 民事诉讼
**问题：** 离职员工把客户名单卖给竞争对手，公司怎么起诉要求赔偿？
**预期领域：** 商事、民事诉讼
**相关法律：** 反不正当竞争法、民事诉讼法
**说明：** 商业秘密侵权 + 诉讼管辖与执行

### 7. 婚姻 + 治安
**问题：** 丈夫长期家暴妻子，女方报警后警方会怎么处理？能申请人身保护令吗？
**预期领域：** 婚姻、治安
**相关法律：** 民法典、治安管理处罚法
**说明：** 家暴属于治安违法行为，人身保护令属于婚姻家事程序

### 8. 监察 + 刑事
**问题：** 政府官员收受企业贿赂为对方谋取项目审批便利，会被怎么处分？构成犯罪吗？
**预期领域：** 监察、刑事
**相关法律：** 监察法、刑法
**说明：** 政务处分 + 受贿罪

### 6. 劳动 + 未成年人 + 民事诉讼
**问题：** 16 岁少年冒用他人身份证入职后遭遇工伤，公司发现后拒绝赔偿并报警，家长该怎么办？

**说明：**

- 未成年人：未成年人劳动权益保护
- 民事诉讼：申请仲裁、起诉举证

## 智能案情分析

> 用户消息被分类为 `intent=analysis` 时，系统启动 LangGraph 多步推理图：
> 案情拆解 → 并行检索（法律关系/证据/维权路径）→ 交叉分析 → 结构化报告。
> 前端显示「案情分析」标签 + 「生成文书」下拉按钮。

### 1. 劳动纠纷 — 违法辞退

**问题：** 我在公司工作了三年，上个月公司以组织调整为由突然辞退我，没有提前通知也没有给补偿金，我该怎么维权？
**预期意图：** analysis
**验证要点：** 输出 4 部分报告（法律关系定性/主张分析/证据缺口/维权路径），报告中应提及劳动合同法相关条款

### 2. 交通事故 — 人身损害

**问题：** 我骑电动车被汽车撞了，交警认定对方全责，我住院花了两万多，对方只肯赔一万，我能起诉吗？需要准备什么证据？
**预期意图：** analysis
**验证要点：** 报告应包含赔偿主张分析、证据清单（医疗费票据/事故认定书/误工证明）、维权路径建议

### 3. 合同纠纷 — 违约赔偿

**问题：** 我跟装修公司签了合同约定两个月完工，结果拖了半年还没做完，质量也有问题，我能要求退款和赔偿吗？
**预期意图：** analysis
**验证要点：** 报告应分析合同违约责任、可主张的赔偿范围、证据缺口（合同/付款凭证/沟通记录）

### 4. 借款纠纷 — 民间借贷

**问题：** 朋友找我借了十万块钱说好三个月还，现在快两年了一直推脱不还，也没有借条只有转账记录，能打赢官司吗？
**预期意图：** analysis
**验证要点：** 报告应分析借贷关系认定、证据效力（转账记录/聊天记录）、诉讼时效

## 诉讼时效计算器

> 用户消息被分类为 `intent=statute` 时，系统调用 LLM 提取时间节点 + 规则引擎计算时效。
> 支持 5 种时效类型：劳动仲裁(1年)、普通民事(3年)、人身损害(3年)、产品质量(2年)、环境污染(3年)。
> 输出包含：时效类型、起算日期、截止日期、剩余天数、是否过期。

### 1. 劳动仲裁 — 在时效内

**问题：** 我今年3月被公司辞退，现在8月了，还能申请劳动仲裁吗？
**预期意图：** statute
**验证要点：** 识别为劳动仲裁（1年时效），输出"还在时效内"，显示剩余天数

### 2. 劳动仲裁 — 已过期

**问题：** 我2023年1月被公司拖欠工资，到现在还没要回来，还能仲裁吗？
**预期意图：** statute
**验证要点：** 识别为劳动仲裁（1年时效），输出"已过期"，显示已超期天数

### 3. 普通民事 — 借款时效

**问题：** 朋友2022年6月借了我五万块，约定一年还，到现在都没还，诉讼时效过了吗？
**预期意图：** statute

## 法律文书自动生成

> 用户消息被分类为 `intent=document` 时，系统进入文书生成链路。
> 当前主线只启用劳动人事争议仲裁申请书；民事起诉状、律师函、合同审查意见暂不作为本轮验收目标。
> 文书以字段抽取 + 缺失字段提示 + 模板预览为主，不让模型自由编造事实。

### 1. 劳动仲裁申请书

**问题：** 帮我写一份劳动仲裁申请书，我被公司违法辞退了，要求赔偿
**预期意图：** document
**验证要点：** 输出格式规范的劳动仲裁申请书，包含申请人/被申请人信息、仲裁请求、事实与理由

### 2. 非劳动文书暂不启用

**问题：** 写一个民事起诉状，对方借了我十万块钱不还
**预期意图：** document / unsupported
**验证要点：** 不强行生成当前未启用的文书类型，可提示暂不支持或引导补充后续规划

## 性能优化与降级

> 以下用例验证 Phase 2 引入的性能优化：并行缓存、查询复杂度路由、流式降级。

### 语义缓存命中

**问题：** 试用期最长多久？
**操作：** 连续发送两次相同问题
**验证要点：**

- 第二次请求日志出现 `[语义缓存] 精确命中`
- 第二次响应明显更快（跳过 RAG 全链路）
- 两次回答内容一致

### 并行缓存超时

**操作：** 首次提问（冷启动，缓存为空）
**验证要点：**

- 日志无 `缓存查找异常` 或 `缓存超时` 报错
- 缓存查找不阻塞 RAG 链执行（两者并行）

### 简单查询模式

**问题：** 试用期最长多久？
**验证要点：**

- 日志出现 `简单查询模式，跳过精排`（无 `[Rerank]` 行）
- 无案例检索日志
- 响应延迟低于复杂问题

### 复杂查询完整流程

**问题：** 劳动仲裁时效和工伤认定时效有什么区别？分别怎么计算？
**验证要点：**

- 日志出现 `[Rerank]` 精排记录
- 走完整 7 步流水线（分类→重写→检索→Rerank→扩展→生成→校验）
- 默认无案例检索；如手动启用案例检索，只应返回高相关案例

### 流式降级

**操作：** 模拟流式接口异常（如后端 LLM 超时）
**验证要点：**

- 前端不显示空白或卡住，自动降级为非流式返回
- 日志出现 `流式降级` 提示
- 用户仍能获得完整回答

### 前端状态持久化

**操作：**

1. 发送几条消息建立会话
2. 刷新浏览器（F5）

**验证要点：**

- 刷新后会话 ID 不变
- 历史消息仍显示（从 localStorage 恢复）
- 切换会话后再刷新，能恢复到上次选中的会话

| `[merge_contexts]` | 多域检索结果合并去重 |
| `[混合检索]` | BM25 + 向量 RRF 融合结果数 |
| `[Rerank]` | Rerank 精排前后数量 |
| `[流式输出]` | LLM 流式回答开始 |
| `[案情分析]` | 案情分析图执行进度 |
| `[时效计算]` | 诉讼时效计算结果 |
| `[文书生成]` | 法律文书生成开始 |
| `[语义缓存]` | 缓存命中/未命中 |
| `[案例库]` | 案例库加载状态（启动时） |
| `SQLite 已加载` | FTS5 案例库就绪 |
| `LanceDB 已加载` | 语义案例库就绪（首次启动显示"从 SQLite 构建中"） |
