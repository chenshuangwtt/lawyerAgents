# 法律顾问 Agent — 扩展与优化规划

## 一、现有代码优化

### 架构层

1. ~~**检索链路可观测性**~~ ✅ 已完成 — Python logging 替代 print()，结构化日志（`[混合检索]`、`[Rerank]`等标记），日志同时写入 stdout + `logs/app.log`（RotatingFileHandler，10MB 轮转）
2. ~~**语义缓存**~~ ✅ 已完成 — 精确匹配（SHA256）+ 语义匹配（embedding 余弦相似度 ≥ 0.92），SQLite 存储，TTL 过期 + LRU 淘汰，前端显示「来自缓存」标识
3. ~~**存储升级**~~ ✅ 已完成 — PostgreSQL + SQLite 双后端，通过 DATABASE_URL 环境变量切换，psycopg2 连接池 + RealDictCursor

### 功能层

4. ~~**分类准确率**~~ ✅ 已完成 — 加权关键词快速分类（≥0.7 置信度直接返回，0ms），低置信度走 LLM 兜底；`classify_by_keywords()` + `classify_question()` 改造
5. ~~**检索质量评估**~~ ✅ 已完成 — 31 个结构化测试用例（单域/多域/案例检索），`scripts/eval_retrieval.py` 批量评估 + 分领域/分类型报告
6. ~~**引用精确到段落**~~ ✅ 已完成 — PARA_PATTERN 匹配"第X条第Y款"，prompt 指示 LLM 精确引用，提取/验证全链路支持款级粒度
7. ~~**多轮对话能力**~~ ✅ 已完成 — LLM 自动提取案情状态（当事人/纠纷/关键事实/阶段），存入 DB，下一轮 prompt 注入，query 重写增强

### 工程层

8. ~~**SSE 断线重连**~~ ✅ 已完成 — `api.js` sendMessageStream 自动重试 3 次（指数退避 1s/2s），无内容才报错；后端日志 warning 兜底
9. ~~**流式进度条**~~ ✅ 已完成 — 后端 pipeline 每步 `time.perf_counter()` 计时，SSE `substep` 事件发送 `elapsed_ms`；前端 MessageBubble 渲染进度时间线（分类→检索→精排→扩展→生成），生成阶段蓝色脉冲动画
10. ~~**配置热更新**~~ ✅ 已完成 — 14 个检索参数白名单，`Settings.update()` 类型安全，`GET/PUT /api/config` 端点

### 优先级

`其余待定`

## 二、进阶计划

### 阶段一：深度分析能力

1. ~~**智能案情分析**~~ ✅ 已完成 — 自动检测案情意图，LangGraph 多步推理（拆解→并行检索→交叉分析→报告），4 部分结构化报告（法律关系/主张分析/证据缺口/维权路径），流式进度展示

**2. ~~诉讼时效计算器~~** ✅ 已完成 — `app/statute.py` 规则计算（劳动仲裁1年/民事3年/人身损害3年等），集成到案情分析报告「维权路径与时间线」，独立时效问答（intent=statute），LLM 提取时间+规则计算

### 阶段二：文书生成

**3. ~~法律文书自动生成~~** ✅ 已完成 — `app/document_generator.py` 模板+LLM填充，4种文书（劳动仲裁申请书/民事起诉状/律师函/合同审查意见），案情分析报告「生成文书」按钮+独立问答（intent=document），POST /api/document 端点

### 阶段三：数据闭环

4. ~~**案例库 + 相似案例检索**~~ ✅ 已完成 — CaseMatch（LeCaRD 刑事数据集）+ SQLite FTS5 + LanceDB 语义检索 + RRF 融合，领域过滤，概览跳过

**5. 用户反馈迭代**
- 收集用户评价（有用/没用/引用有误）
- 自动标注低质量回答
- 定期优化 prompt 和检索参数

### 建议优先级

**先做 1（案情分析）** — 和现有 RAG 链路一脉相承，LangGraph 自然扩展，演示效果从"能回答法律问题"升级到"能分析法律案件"
