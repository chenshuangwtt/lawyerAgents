# RAG Eval Expansion Plan

当前 `questions.jsonl` 有 30 条样本，其中 13 条是多法律样本。下一阶段目标是扩到 80-100 条，优先增加跨法律、跨领域、强程序法依赖的问题，避免在当前 30 条上过拟合。

## 目标配比

- 单法律基础问题：20-30 条
- 双法律问题：30-40 条
- 三法律及以上问题：20-30 条
- 程序法参与问题：15-25 条

## 优先补充类型

- 劳动 + 商事 + 刑事：离职、客户名单、删库、挪用资金、竞业限制、商业秘密。
- 婚姻 + 治安 + 刑事/刑诉：家暴报警、伤情鉴定、人身安全保护令、拘留、证据固定。
- 电诈 + 刑法 + 刑诉 + 反诈法：刷单、跑分、银行卡冻结、报案证据、追赃。
- 民法典 + 民诉 + 仲裁：借贷证据、合同管辖、仲裁协议、诉讼时效。
- 公司法 + 民法典 + 反不正当竞争：空壳主体、股东责任、客户名单、虚假宣传。
- 未成年人 + 刑法 + 民法典：校园伤害、监护责任、刑责年龄、赔偿。

## 每条样本字段

必填：

- `id`
- `question`
- `domain`
- `expected_laws`
- `type`

建议逐步补充：

- `ground_truth`
- `expected_articles`
- `difficulty`
- `notes`

## 通过标准

扩容后先只看召回：

- Final Contexts target hit >= 95%
- 多法律样本 law all hit >= 90%
- 数据覆盖缺口必须单独列出，不能算作算法漏召

召回稳定后，再补 `ground_truth` 并接 RAGAS 评估生成质量。
